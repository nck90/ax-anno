# 법원경매공고 편집 자동화 스펙 v1.0

> 원본 법원 원고 → 신문 게재용 편집본 자동 변환 규칙  
> 기준: 편집기준(2023.12.04) + 실무 사례 분석 통합

---

## 0. 용어 정의

| 용어 | 설명 |
|------|------|
| **원본** | 법원에서 제공한 HWP/HWPX 원고 파일 |
| **편집본** | 신문 게재를 위해 축약·정리한 결과물 |
| **소재지** | 편집본의 "소재지 및 면적[㎡]" 칸 |
| **비고** | 편집본의 "비 고" 칸 |
| **지목** | 토지의 용도 (대, 답, 전, 임야, 공장용지 등) |
| **제시외** | 등기부에 없으나 현장에 존재하는 건물/수목 등 |
| **동소** | 같은 동/리 내 다른 번지를 가리키는 축약어 |
| **목록** | 같은 사건 내 여러 필지의 순서 번호 |

## 1. 테이블 구조 변환

### 1.1 컬럼 변환
```
원본 (7컬럼): 사건번호 | 물건번호 | 용도 | 소재지 | 상세내역(구조및면적) | 감정평가액/최저매각가격 | 비고
편집본 (6컬럼): 사건번호 | 물건번호 | 소재지 및 면적[㎡] | 용도 | 감정평가액/최저매각가격[단위:원] | 비고
```

**변환 규칙:**
- `소재지` + `상세내역` → `소재지 및 면적[㎡]` (합침)
- `용도` 컬럼 위치: 소재지 다음으로 이동
- `(단위:원)` → 감정평가액 칸 하단에 표기

### 1.2 카테고리 분류

물건을 5개 카테고리로 분류하여 별도 테이블로 분리:

| 카테고리 | 분류 조건 (원본 용도 필드 기준) |
|---------|--------------------------|
| `[아파트]` | 용도 = "아파트" |
| `[단독주택,다가구주택]` | 용도에 "단독주택" 또는 "다가구주택" 포함 |
| `[대지/임야/전답]` | 용도 = "전답" 또는 "임야" 또는 "대지" |
| `[상가,오피스텔,근린시설]` | 용도에 "상가", "오피스텔", "근린시설" 포함 |
| `[기타]` | 위 어디에도 해당하지 않음 |

**규칙:**
- 카테고리 간 표 순서는 자유 (고정할 필요 없음)
- 카테고리 내 사건번호는 **오름차순 정렬** (연도→번호 순)
- 하나의 사건에서 물건별 용도가 다르면 **카테고리를 분리** (예: 물건1=기타, 물건2=임야 → 각각 다른 테이블)

### 1.3 사건번호 정렬 알고리즘
```
정렬키 = (연도: int, 번호: int)
"2025타경5318" → (2025, 5318)
"2022타경52669" → (2022, 52669)
오름차순: (2022, 52669) < (2025, 76) < (2025, 5318)
```

## 2. 소재지 변환

### 2.1 광역시/도 삭제
```
삭제 목록: 충청북도, 충청남도, 경상북도, 경상남도, 전라북도, 전라남도, 
          전북특별자치도, 경기도, 강원도, 강원특별자치도,
          서울특별시, 부산광역시, 대구광역시, 인천광역시,
          광주광역시, 대전광역시, 울산광역시, 제주특별자치도
유지: 세종특별자치시 (삭제 안 함)
```

**알고리즘:** `text.replace(/^(충청북도|충청남도|...) /, '')`

### 2.2 소재지 + 상세내역 합치기

| 상세내역 패턴 | 변환 규칙 | 예시 |
|-------------|---------|------|
| `지목 면적㎡` | 지목 삭제, 면적만 소재지에 추가 | `대 178㎡` → 소재지에 `178㎡` 추가, 용도="대" |
| `지목 면적㎡ 제시외 ...` | 지목 삭제 + 제시외 간략화 | `답 887㎡ 제시외 -수목 일괄` → `887㎡ 제시외 수목일괄` |
| `지목 면적㎡ (지분정보)` | 지목 삭제 + 지분정보를 [대괄호]로 | `임야 131435㎡ (고우열...)` → `131435㎡ [고우열...]` |
| 건물: `구조 면적㎡` | 구조 삭제, 층+면적만 | `철근콘크리트구조 126.901㎡` → `126.901㎡` |
| 건물: 다층 | 구조 삭제, 층별 면적 | `1층 104.87㎡ 2층 95.04㎡` → `1층104.87㎡ 2층95.04㎡` |

### 2.3 건물 구조 설명 삭제

**삭제 키워드 목록** (정규식):
```
/(철근콘크리트|경량철골|시멘트?벽돌|연와|조립식|판넬|강판|시멘벽돌|콘크리트평슬래브|
  강파이프|칼라강판|컬러강판|샌드위치패널|아스팔트싱글|기와형가판|스라브|슬래브|
  판넬지붕|기타지붕|썬라이트지붕|아크릴지붕)구?조?\s*/g
```

**삭제하되 유지할 것:**
- 층수 (단층, 1층, 2층...)
- 면적 (N㎡)
- 용도명 (단독주택, 근린시설, 공장 등)

### 2.4 건물 용도명 축약

| 원본 | 축약 |
|------|------|
| 제2종근린생활시설 | 근린시설 |
| 제1종근린생활시설 | 근린시설 |
| 위험물저장 및 처리시설 | 위험물저장및처리시설 |
| 제조업소 | [제조업소] |

### 2.5 동일 면적 층수 합치기
```
"1층 73.88㎡ 2층 73.88㎡" → "1,2층각73.88㎡"
```
**알고리즘:** 연속된 층의 면적이 동일하면 `N,M층각X㎡`로 합침

### 2.6 "동소" 축약

**조건:** 같은 사건 내에서 연속되는 필지의 주소가 "숫자 앞까지 동일"할 때

**알고리즘:**
```
기준주소 = 첫 번째 필지의 "시/군 + 구/동/읍/면/리" 부분
이후 필지의 기준주소 == 이전 필지의 기준주소 → "동소 {번지}"
```

**예외:**
- 도로명 주소(~로, ~길)는 동소 적용 **안 함** (전체 주소 기재)
- 기준주소가 변경되면 동소 리셋

**예시:**
```
제천시 강제동 136-3 → 제천시 강제동 136-3 (전체)
제천시 강제동 산16-26 → 동소 산16-26
제천시 강제동 132-3 → 동소 132-3
제천시 북부로 2365 → 제천시 북부로 2365 (도로명이므로 동소 안 함)
제천시 강제동 136-7 → 동소 136-7 (도로명 뒤에도 강제동 기준 동소 재개)
```

### 2.7 괄호 변환
```
() → []  (모든 소괄호를 대괄호로)
```

### 2.8 지분 정보

**규칙:** 지분내용은 반드시 소재지에 [대괄호]로 기재. 비고에 넣으면 안 됨.

**변환:**
```
"공유자 백종필 지분 2분의 1" → "[공유자백종필1/2지분]"
"고우열 지분 전부 131435분의 16520" → "[고우열16520/131435지분전부]"
"채무자 김형래 지분 10분의1 전부" → "[채무자김형래1/10지분전부]"
```

**분수 역전 알고리즘:**
```
"N분의 M" → "M/N"  (분모/분자 → 분자/분모 + 슬래시)
"N분의M" → "M/N"
```

### 2.9 띄어쓰기 규칙

| 위치 | 규칙 | 예시 |
|------|------|------|
| 주소 (시/군/구/동/리/번지) | **띄어쓰기 유지** | `제천시 봉양읍 연박리 365` |
| 동, 층, 호 | 원칙적 띄어쓰기, 줄수 줄면 붙이기 | `101동 11층1101호` |
| 층 + 면적(㎡) | **붙여쓰기** | `1층104.87㎡` |
| [대괄호] 안 내용 | **붙여쓰기** (공백 삭제) | `[농지취득자격증명요[소관행정기관에...]]` |
| 제시외 설명 | **붙여쓰기** | `제시외 창고등38.2㎡` |

## 3. 제시외 변환

### 3.1 제시외 건물 합산

**알고리즘:**
1. 상세내역에서 "제시외" 이후의 모든 면적(㎡) 값을 합산
2. 대표 건물명 = 제시외 항목 중 **글자수가 가장 짧은 것**
3. 항목이 2개 이상이면 "등" 추가

```
입력: "제시외 -비가림막 22.4㎡ 제시외건물㉠-비가림막 10㎡"
대표명: "비가림막" (3글자, 둘 다 같음)
합산: 22.4 + 10 = 32.4
출력: "제시외 비가림막등32.4㎡"
```

```
입력: "제시외 -사무소 9.50㎡ -화장실,보일러실 4.3㎡ -창고 6.6㎡ -견사 14.26㎡ -전실 4.44㎡"
대표명: "전실" (2글자, 가장 짧음)
합산: 9.50 + 4.3 + 6.6 + 14.26 + 4.44 = 39.1
출력: "제시외 전실등39.1㎡"
```

### 3.2 제시외 기계기구

```
"기계기구 셀프주유기 外 6식" → "기계기구6식"
(n식 단위는 합산, ㎡와 별도 표기)
```

### 3.3 제시외 수목

```
"수목 수목 일괄㎡" → "수목일괄"
"과수목 감나무,매실 1식" → "과수목 감나무,매실1식"
```

### 3.4 제시외 면적 소수점
```
합산 결과 소수점: Math.round(total * 100) / 100
정수면 정수로 표기: 32.0 → 32
```

## 4. 비고 변환

### 4.1 최우선 규칙: 배치 순서
```
비고 맨 앞: "일괄매각" 또는 "지분매각"
[중복] 비고 옮길 시: "2022타경55682[중복].일괄매각...."  (사건번호 맨 앞)
```

### 4.2 공백 제거
비고 내용은 **모든 공백 삭제 후 붙여쓰기**가 원칙.

### 4.3 농지취득자격증명

| 원본 | 변환 |
|------|------|
| `농지취득자격증명 제출요(미제출시 보증금 미반환, 소관 행정기관에 농지취득자격증명 발급 여부 사전 문의 후 입찰 요망)` | `농지취득자격증명요[소관행정기관에발급여부사전문의후입찰요]` |
| `농지취득자격증명 제출요(미제출시 보증금 미반환, 소관 행정기관에 증명 발급 여부 사전 문의 후 입찰 요망)` | `농지취득자격증명요[소관행정기관에증명발급여부사전문의후입찰요]` |
| `[농지법 제8조 제1항 제1호에 따라 농지취득자격증명원 필요하지 않음]` | `농지취득증명원불요` |

### 4.4 매각물건명세서 관련

| 원본 패턴 | 변환 |
|---------|------|
| `매각물건명세서, 부동산현황조사보고서, 감정평가서 등을 통하여 철저한 사전 조사 후 입찰자 본인의 책임하에 입찰 요망` | `매각물건명세서,부동산현황조사보고서,감정평가서등통해철저한사전조사후입찰자본인책임하입찰요` |
| `~등을 참고하고 현지 답사 등을 통하여 철저히~` | `~등참고,현지답사등통해철저히~` |
| `~등을 참고하여 조사된 임차인의 권리관계를 철저히 조사한 후~` | `~등참고해조사된임차인의권리관계를철저히조사한후~` |

### 4.5 특별매각조건/공유자우선매수

```
"특별매각조건 있음(공유자 우선매수권행사는 1회로 제한함)" 
→ "공유자우선매수권1회제한"  (특별매각조건~ 삭제)

"지분매각, 공유자의 우선매수신고는 1회에 한하여 행사할 수 있음"
→ "지분매각.공유자우선매수신고1회제한"
```

### 4.6 조사 축약

| 패턴 | 변환 |
|------|------|
| `~바람` | `~요` |
| `~임` (문장 끝) | 삭제 (`맹지임`→`맹지`, `이용중임`→`이용중`) |
| `~요망` | `~요` |
| `본인의 책임하에` | `본인책임하` |
| `더덕및나물` | `더덕등` |
| `조경용으로조성된연못` | `조경용조성연못` |

### 4.7 삭제 규칙

| 대상 | 삭제 규칙 | 예외 |
|------|---------|------|
| 상호명 | 삭제 | 주소의 일부인 경우 유지 (예: `[강제동,상일주유소]`) |
| 특정인 이름 | 삭제 | **지상권자/공유자 등 법적 관계자는 유지** |
| 날짜 | 삭제 | **법적 행위일(동의서 제출일 등)은 유지** |
| 건축자재 | 삭제 | 비고에 특별 언급 있으면 유지 |
| `공부상 지목이 ~이나` | 삭제 | 현황 정보만 남김 |

**날짜 삭제 알고리즘:**
```
삭제: "2020.1.13.자 현황보고서 제출됨" → "현황보고서제출됨"
유지: "을구5번 지상권자...로부터 '최고가매수인이...' 동의서가 제출됨" 
      → 날짜는 법적 행위 맥락에서 유지 가능하나, 
         편집기준은 삭제 원칙이므로 삭제 우선.
         AI가 "동의서 제출" 문맥에서 날짜를 삭제해도 의미 전달 가능하면 삭제.
```

### 4.8 문장 마침표

```
규칙: 문장 구분은 "."으로. 마지막 문장의 "."는 삭제.
예: "일괄매각.제시외건물ㄱ~ㄷ매각포함.주의사항:매수인에게대항할수있는임차인있을수있음"
    (마지막에 "." 없음)
```

### 4.9 연속 숫자 축약

```
"목록 1,2,3" → "목록1~3"
"목록 6,7,8" → "목록6~8"
```

```python
    note_lines = ceil(len(condensed_note) / 20)

        move_to_location(농지취득자격증명_부분)

    # 5. 하위필지가 있으면 → 마지막 필지 소재지에 [물건번호N:...] 형태로
    if sub_parcels:
```

## 6. 용도 결정

### 6.1 기본 규칙
- 원본 상세내역의 지목/용도를 그대로 사용
- 원본 용도가 "기타"면 상세내역에서 추출

### 6.2 용도 추출 우선순위
```
1. 상세내역 첫 단어가 지목이면 → 해당 지목 (대, 답, 전, 임야...)
2. 상세내역에 건물 용도명이 있으면 → 해당 용도 (공장용지, 주유소용지...)
3. 건물의 경우 → 건물 유형 (단독주택, 근린시설, 공장...)
4. 확인 불가 → "기타"
```

### 6.3 용도가 소재지에 중복되면 소재지에서 삭제
```
소재지: "... 단독주택 1층73.62㎡ ..."  용도: "단독주택"
→ 소재지에서 "단독주택" 삭제: "... 1층73.62㎡ ..."
```

### 10.3 줄수 균형

**자동화 알고리즘:**
```python
CHARS_PER_LINE_LOC = 25   # 소재지 칸 한 줄 약 25자 (7단 신문 기준)
CHARS_PER_LINE_NOTE = 20  # 비고 칸 한 줄 약 20자

def calc_lines(text, chars_per_line):
    return max(1, ceil(len(text) / chars_per_line))

def balance_lines(location, remarks, sub_count):
    loc_lines = calc_lines(location, 25) + sub_count
    note_lines = calc_lines(remarks, 20)
    
    if note_lines > loc_lines + 2:
        # 비고에서 이동 가능한 부분 추출
        movable = extract_movable_parts(remarks)  # 농지, 매각물건명세서 등
        for part in movable:
            remarks = remarks.replace(part, '')
            location += f" [{part}]"
            # 재계산
            if calc_lines(remarks, 20) <= calc_lines(location, 25) + 1:
                break
    
    return location, remarks
```

## 11. 검증 체크리스트 (자동화 후 자동 검증)

| # | 검증 항목 | 방법 |
|---|---------|------|
| 1 | 모든 사건번호가 보존되었는가 | 원본 사건번호 집합 == 편집본 사건번호 집합 |
| 2 | 모든 물건번호가 보존되었는가 | 사건별 물건번호 리스트 비교 |
| 3 | 감정평가액/최저매각가격이 정확한가 | 원본 vs 편집본 1원 단위 비교 |
| 4 | 면적(㎡) 값이 보존되었는가 | 원본 면적 집합 ⊆ 편집본 텍스트 |
| 5 | 감정평가서 참조 문구가 유지되었는가 | 원본에 있으면 편집본에도 있어야 함 |
| 6 | 카테고리 내 사건번호가 오름차순인가 | 정렬 검증 |
| 7 | 비고→소재지 이동이 [대괄호] 안인가 | 정규식 검증 |
| 8 | 마지막 문장에 "."가 없는가 | 비고 끝 문자 검증 |
| 9 | 광역시/도가 삭제되었는가 | 삭제 목록의 문자열이 편집본에 없어야 함 |
| 10 | 건물 구조 키워드가 삭제되었는가 | 구조 키워드가 편집본에 없어야 함 |
